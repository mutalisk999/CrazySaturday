# 疯狂星期六 - 功能迭代计划

## 需求概述

根据 `prompts/feat-new.md` 文档，本次迭代需要实现以下功能：

1. **移除"设定球桌阈值"界面**：不在主界面显示该子界面
2. **移除自动减桌机制**：不再通过选手数量自动决定球桌数量和触发撤台操作
3. **手动撤台功能**：在每个球台上添加"撤台"按钮，用户手动触发撤台操作
4. **重新开始比赛对话框**：点击重新开始比赛按钮时，弹出对话框让用户选择初始使用的球桌数量
5. **状态机兼容**：所有修改都需要考虑状态回退和状态前进的情况

### 阶段二：Game模型修改

#### 任务2.1：修改球台初始化逻辑
- **文件**：`models/game.py`
- **位置**：`setup_tables()` 方法（约第77-88行）
- **修改内容**：

  - 不再根据选手数量自动计算球台数量
  - 使用固定的球台使用顺序：`[2, 3, 4, 5, 6, 7, 8, 9, 1]`
  - 根据用户选择的数量初始化球台

#### 任务2.2：移除自动减桌逻辑
- **文件**：`models/game.py`
- **修改内容**：
  - 移除或禁用 `check_table_reduction()` 方法中的自动减桌逻辑
  - 保留 `tables_to_close` 和 `closing_tables` 字典，用于手动撤台
  - 移除 `calculate_required_tables()` 方法的调用（保留方法以备后用）

#### 任务2.3：添加手动撤台方法
- **文件**：`models/game.py`
- **新增方法**：`manual_close_table(table_id: int)`
- **功能**：
  - 标记指定球台进入需要撤台的状态
  - 后续撤台逻辑维持不变（与之前的自动撤台逻辑一致）
  - 保存状态到状态历史

#### 任务2.4：修改 `start_game()` 方法
- **文件**：`models/game.py`
- **位置**：`start_game()` 方法（约第108-139行）
