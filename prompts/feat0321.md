# 固定参赛选手

* "加载历史状态"按钮左边需要新增一个按钮"固定参赛选手"，点击后进入固定参赛选手界面（这个界面上有一个返回主界面的按钮，点击后可以返回到原先的主界面）
* 用户可以在该界面添加、删除、修改固定参赛选手。参赛选手的基本属性信息包含但不仅限于:
    - 姓名 （必填，唯一不能重复，姓名的校验限制和主界面上的校验限制相同）
    - 联系方式 （必填）
    - 邮箱 （选填）
    - 家庭住址  （选填）
    - 初始HP （必选，参考主界面上可选的2-8的checkbox组方式的供选择的初始HP）
* 固定参赛选手的属性信息需要持久化保存在json文件中，文件名固定为"fixed_participants.json"
* 在主界面上添加新选手时，原先输入选手名的文本框需要被替换成支持文本输入的下拉框替换，下拉框中显示所有固定参赛选手的姓名，如果固定参赛选手的名字被选中，他的属性信息HP会自动填充到主界面上初始HP设置上。


# 支持动态调整桌数和选手数的对应关系

* 当前选手数和桌数有固定的对应关系，具体可以参考 models.py中的成员变量 self.table_thresholds以及成员方法calculate_required_tables，其中的6,10,14,17,20,24,27,31就对应的需要的配置项。在self.table_thresholds中就代表>6个人就需要2张桌子，在calculate_required_tables中就代表<=6个人需要1张桌子，以此类推。
* 在"固定参赛选手"按键左侧新增一个按钮"设定球桌阈值"，点击后进入设定球桌阈值界面（这个界面上有一个返回主界面的按钮，点击后可以返回到原先的主界面）
* 用户可以在这个界面修改1-8张桌子的选手的阈值，默认就是6,10,14,17,20,24,27,31，并且需要校验都是正数，且后一个数值需要大于前一个数值。
* 球桌阈值需要持久化保存在json文件中，文件名固定为"table_thresholds.json"
* 阈值修改后，历史状态文件都无法支持，需要删除所有历史状态文件（文件名固定为"game_states.json"
），这一点需要在球桌阈值界面有相关的文案重点提示。
* self.table_thresholds以及calculate_required_tables中的相关数值都应该通过球桌阈值文件加载获取，如果没有获
取成功，则使用默认值6,10,14,17,20,24,27,31
