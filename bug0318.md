# 新增已离场桌台队列
需要新增一个已离场桌台队列
- 某桌有人离场就增加桌号到尾部，如果桌号已经在队列里了，就不重复添加。桌号被标记为需要撤桌的桌号如果有人离场，不需要添加。
- 从队列头部开始选择桌号，安排场外的选手上桌。如果挑战者和候选者都为空就安排挑战者和候选者，如果挑战者不为空但是候选者为空就安排候选者。如果安排满员了，则需要删掉这个桌号。如果没有满员安排，就需要继续保留这个桌号，等待下次安排。如果队列头部的桌号被标记为需要撤桌，则不需要安排，直接删除这个桌号。这样一直循环下去，直到尽量给所有场外候选者都能安排到球桌上去。

# 离场逻辑
由”本局离场“按键触发，流程如下：
- 在一张球桌上比赛，失败的人离场，胜利的人留下。如果获得胜利的是挑战者，则用户成为擂主，如果获得胜利的是擂主，就擂主不变。球桌有候补者，就变成挑战者，如果球桌没有替补者就不处理这个动作。对于离场的选手HP需要减1，后根据减1后的HP值，如果HP>0就进场外候补区等待，如果HP<=0就进淘汰区（有个特殊场景，如果球桌是标记为撤桌状态的话，如果同时球桌候补为空，比赛的胜利者也要一起离场，进场外候补区，然后将球桌真正回收）

# 上场逻辑
在离场触发的流程完成后触发上场流程，流程如下：
- 从已离场桌台队列头部开始选择桌号，安排场外候选的选手上桌。如果挑战者和候选者都为空就安排挑战者和候选者，如果挑战者不为空但是候选者为空就安排候选者。如果安排满员了，则需要删掉这个桌号。如果没有满员安排，就需要继续保留这个桌号，等待下次安排。如果队列头部的桌号被标记为需要撤桌，则不需要安排，直接删除这个桌号。这样一直循环下去，直到尽量给所有场外候选者都能安排到球桌上去。
